# KiriLog 開発・設計マスターガイド（決定版）

このドキュメントは、KiriLogの**「設計の本質（なぜ？）」**と**「具体的な実装（どうやって？）」**の両方を網羅した、開発者向けの完全ガイドです。

---

## 1. 全体構造：レストランに例えた設計思想

KiriLogのコードは、プロが大規模開発でも使う **「関心の分離（Separation of Concerns）」** という考え方で整理されています。これをレストランに例えると分かりやすくなります。

### 📁 src/App.tsx：ホールのマネージャー（客席管理）
アプリの「司令塔」です。
- どの画面（入力、履歴、管理）を出すかを判断し、全体の案内（ナビゲーション）を管理します。
- `useState` という**「短期記憶」**を使い、「いま、お客さんはどのテーブル（画面）にいるか？」を絶えず把握しています。

### 📁 src/features/：各テーブル（各機能の画面）
役割ごとに個別の部屋（コンポーネント）になっています。
- **input/**: 注文票を書く場所（データの入力）。
- **records/**: 過去の注文履歴を見る場所。Markdownへの書き出しもここが行います。
- **data-management/**: レジや帳簿の管理。CSVでのバックアップを担当します。

### 📁 src/lib/：厨房のレシピ（裏方ロジック）
見た目（UI）を持たない、純粋な「計算・保存」のロジック層です。
- **storage.ts**: データの保存、時間の重なりチェックなど、アプリが正しく動くための「計算ドリル」が集まっています。

## 2. 実装のディープダイブ（具体的な仕組み）

ここからは、コードの中で特筆すべき重要なロジックを解説します。

### A. データの型定義 (`src/lib/storage.ts`)
`LogEntry` というインターフェースでデータの形を定義しています。
- **`id`, `date`, `startTime`, `endTime`, `content`, `memo`**
- これにより、アプリのどこでデータを使っても「このデータには何が入っているか？」をTypeScriptが厳密にチェックし、凡ミスを防いでくれます。

### B. 重複判定の数学ロジック (`findOverlapping`)
新しい記録を保存する際、既存の記録とぶつからないかを計算しています。
- 原理: `(開始時間A <= 終了時間B) && (終了時間A >= 開始時間B)`
- この条件を満たす場合、その時間は重なっていると判断し、ユーザーに確認ダイアログ（上書きするか、追記するか）を表示します。

### C. 時間の自動セット (`src/features/input/HourlyForm.tsx`)
`useEffect` と `date-fns` ライブラリを使い、起動時に「直近のキリが良い1時間」を計算します。
1. 現在時刻（例: 14:15）を取得（`now`）。
2. `subHours(now, 1)` で1時間戻す（13:15）。
3. `startOfHour` で秒や分を削ぎ落とし、`13:00` を作る。これが開始時刻になります。

### D. データの書き出し（MarkdownとCSV）
- **Markdown出力 (`RecordsList.tsx`)**: 記録の配列を `map` 関数でループさせ、パイプ記号（`|`）を使って「文字列のパズル」を組み立てます。最後は `navigator.clipboard` でクリップボードに送ります。
- **CSV出力 (`DataManagement.tsx`)**: 全データをカンマ区切りのテキストにし、`Blob`（データのかたまり）を作成。それを疑似的な「ダウンロードリンク」に流し込むことで、ファイル保存を実現しています。

## 3. なぜ「React + Tailwind CSS v4」なのか？

### なぜ React なのか？
従来のプログラミング（jQueryなど）は、「記憶を変えたら、画面のここも変えろ」という2回の手間が必要でした。Reactは**「記憶（State）が変われば、勝手に画面も最新になる」**という仕組みのため、複雑なアプリでもバグが激減します。

### なぜ Tailwind CSS v4 なのか？
最新のv4は、設定ファイル（`tailwind.config.js`）を書く手間をゼロにしました。
CSSファイルに `@import "tailwindcss";` と書くだけで、数千種類のデザインパーツが使えるようになります。これにより、開発者は「名前を付ける」苦労から解放され、直感的にデザインを組めるようになっています。

## 4. あなたがカスタマイズするための指針
何か機能を追加したい時は、以下の手順を思い出してください。

1. **データ自体を増やしたい**: `storage.ts` の `LogEntry` を更新。
2. **新しい計算を加えたい**: `storage.ts` に新しい関数を追加（厨房のレシピを増やす）。
3. **新しい画面を出したい**: `App.tsx` のナビゲーションに新しいボタンを追加。

このマスターガイドは、KiriLogが単なるツールではなく、あなたがWeb開発を学ぶための**「生きた教材」**になるように設計されています。
